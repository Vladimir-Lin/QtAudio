Examples : Full duplex

Full duplex is to direct input device into output device on the same Host API.  Some Host APIs do not support such function.  Full duplex works only between input device and output device on the same Host API.  If you want to redirect between different devices on different Host APIs, you will have to use "Redirect" function.

Because redirect function is a good candidate for full duplex, and full duplex results in the codes of underlying Host API very complicated and hard to maintain its consistency.  In the near future, full duplex may highly possible to be depreciated.  We suggest you not to use this function if possible.

To enable full duplex, just set up inputParameter and outputParameter at the same time you open a stream.  Keep this in mind, a full duplex must use the same Host API, and not all Host API support full duplex.

rt = Open ( &stream           ,
            &inputParameters  ,
            &outputParameters ,
            sampleRate        ,
            frameCount        ,
            streamFlags       ,
            conduit         ) ;
